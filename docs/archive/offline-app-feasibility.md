# Vector Field Lab — 离线独立运行方案评估

**日期**：2026-06-08  
**项目类型**：纯前端 React + Canvas 可视化应用（无服务端依赖）  

---

## 1. 当前环境限制总结

| 方式 | 限制 |
|------|------|
| SCUNET 校园网直连 | ❌ AP 隔离，不可用 |
| 手机热点 | ⚠️ 需每次手动开热点 + 启动服务；两台设备不能离太远 |
| USB 网络共享 | ⚠️ 需数据线，手机插着电脑不方便移动 |
| localhost.run 隧道 | ❌ 不稳定，免费域名回收，运营商可能拦截 |
| Tailscale | ⚠️ 需注册账号 + 两边装客户端，但仍需电脑开机 |

**核心矛盾**：所有方案都要求电脑开机 + 启动 `npm run preview:lan`，本质上还是「手机远程访问电脑」。

---

## 2. 方案评估：把项目变成可独立运行的 App

### 方案 A：PWA（渐进式 Web 应用） ⭐ 推荐

**原理**：利用浏览器原生能力，网页可「安装」到手机/电脑桌面，支持离线运行。

| 维度 | 评估 |
|------|------|
| **当前进度** | 🟢 80% 完成 — manifest ✅ / service-worker ✅ / 离线策略 ✅ |
| **还需做什么** | 注册 SW + 生产构建验证 + 打包说明 |
| **Android 体验** | Chrome → 菜单 →「添加到主屏幕」→ 桌面图标，无地址栏，全屏 |
| **Windows 体验** | Edge/Chrome → 地址栏右侧安装图标 → 独立窗口 |
| **离线能力** | 首次加载后完全离线可用，Canvas/KaTeX/React 全部缓存 |
| **分发方式** | 发文件夹/压缩包即可，手机浏览器打开 `index.html` |
| **工作量** | 🕐 1 小时（注册 SW + 构建验证 + README） |
| **需要用户做的事** | 手机上点「添加到桌面」 |

**SW 已有策略分析**：
```
cache-first    → 脚本/样式/字体/图片   （加载过一次就永远本地读）
network-first  → 导航请求/HTML         （有网用最新，没网用缓存）
版本清理       → 升级 SW 版本号自动清旧缓存
```

### 方案 B：Capacitor（打包为 Android APK）

**原理**：把 Web 应用塞进原生 WebView，生成 `.apk` 安装包。

| 维度 | 评估 |
|------|------|
| 安装体验 | ✅ 跟普通 App 一样，桌面图标，无需浏览器 |
| 离线能力 | ✅ 全部内嵌，完全离线 |
| 分发方式 | 传 `.apk` 文件，手机直接安装 |
| **环境要求** | ❌ 需装 Android Studio（~2GB）+ JDK + Gradle |
| **构建复杂度** | ❌ 有平台兼容问题，首次配置常踩坑 |
| **升级维护** | ⚠️ 改代码→重新构建→重新传 APK→重新安装 |
| **工作量** | 🕐 首次 4-6 小时，后续每次构建 ~5 分钟 |

### 方案 C：Tauri（打包为 Windows 桌面应用）

| 维度 | 评估 |
|------|------|
| 包体积 | ✅ 极小（~5MB） |
| 安装体验 | Windows 原生 `.exe` |
| **Android 支持** | ❌ 不支持（实验阶段） |
| 工作量 | 🕐 2-3 小时 |

## 3. 推荐路径

### 🥇 首选：完善 PWA（零额外依赖，1 小时内完成）

项目已经做了 80%！只差最后一步——在 `index.html` 里注册 service worker，然后构建验证。

完成后效果：
```
手机连热点 → 打开 http://192.168.137.1:4173/
         → Chrome 弹出「添加到主屏幕」
         → 安装后变成桌面图标 App
         → 以后打开直接运行，不需要连热点
```

甚至可以把 `dist/` 文件夹直接传到手机上，用 `file://` 协议打开也能用（需调整 `base` 路径）。

### 🥈 备选：Capacitor APK

如果对「桌面图标 + 无浏览器痕迹」有硬性要求，且愿意花半天配置 Android SDK。

### 🥉 不推荐：Tauri

解决不了手机访问的问题，跟目标不匹配。

## 4. PWA 立即行动计划

1. 在 `index.html` 中添加 SW 注册脚本（2 行代码）
2. `npm run build` 构建生产版本
3. `npm run preview:lan` 验证 PWA 安装和离线可用
4. 更新 README 添加「手机安装」说明
5. 打一个 `dist.zip` 作为可分发离线包
