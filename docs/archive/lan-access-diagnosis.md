**项目**：Vector Field Lab  
**日期**：2026-06-08  
**目标**：手机/平板通过局域网访问电脑上的 Vite 预览服务 (端口 4173)  
**结论**：❌ 校园网 SCUNET 无法实现设备间通信  

---

| 网络层 | 电脑 SYN-ACK 返回手机 | ❌ 被阻断 | 连接始终卡在 `SYN_RECEIVED`，未进入 `ESTABLISHED` |
| 浏览器 | 错误信息演进 | — | `TIMED_OUT` → `ABORTED`（详见下方） |

---

## 3. 错误演进与含义

| 阶段 | 错误 | 含义 |
|------|------|------|
| 初始 | `ERR_CONNECTION_TIMED_OUT` | 手机 SYN 包无法到达电脑（单向不通） |
| 修复 Vite host 后 | `ERR_CONNECTION_ABORTED` | SYN 可达，但三次握手无法完成，Vite 超时断开 |
| 重启 Vite 后 | 连接卡在 `SYN_RECEIVED` | 握手进行到第二步后卡死，确认**返回路径被阻断** |

三次握手过程：
```
手机 ──SYN──→ 电脑   ✅ 到达
手机 ←─SYN-ACK─ 电脑   ❌ 被校园网丢弃
手机 ──ACK──→ 电脑   永不发生
```

---

## 4. 根因分析

### 根本原因：校园网 AP/客户端隔离（AP Isolation / Client Isolation）

SCUNET 作为开放式校园 Wi-Fi，部署了**接入点隔离**安全策略：

- 同一 AP 下的无线客户端**不允许直接通信**
- 每个设备只能与网关/互联网通信，设备间流量被 AP 或控制器丢弃
- 部分实现中 SYN 包可穿透（防火墙状态表中的短暂窗口），但 SYN-ACK 被拦截

### 为何排除了其他可能

| 被排除的原因 | 排除依据 |
|-------------|----------|
| Windows 防火墙 | 入站规则已添加，SYN 包成功到达电脑 |
| Vite 配置错误 | `host: '0.0.0.0'` + `allowedHosts: true`，本地 200 OK |
| 手机浏览器问题 | 换了 Edge 浏览器，现象一致 |
| 端口被占用 | `netstat` 确认 4173 仅 Vite 占用 |
| 子网不同 | 电脑与手机均在 `10.132.248.0/21` 内 |

---

## 5. 可行替代方案

| 方案 | 操作步骤 | 优点 | 缺点 |
|------|---------|------|------|
| **① 电脑开移动热点** | Win+I → 网络 → 移动热点 → 开启；手机连热点 | 完全绕过校园网，即开即用 | 电脑需支持热点功能 |
| **② USB 网络共享** | 数据线连接手机与电脑 → 手机开启「USB 网络共享」 | 有线直连，最稳定 | 需要数据线 |
| **③ 便携路由器** | 额外硬件创建独立 Wi-Fi 子网 | 多设备同时访问 | 需额外购买硬件 |
| **④ SSH 反向隧道** | 使用 localhost.run / ngrok 等公网隧道 | 无需改动网络 | 不稳定（免费隧道会回收域名）、延迟高 |

### 推荐方案

**首选方案 ①（Windows 移动热点）**——无需额外设备，30 秒配置完成，网络完全独立于 SCUNET。

Vite 和防火墙配置均已就绪，手机连上热点后直接访问 `http://<新IP>:4173/` 即可。

---

## 6. 总结

- **Vite 服务端完全正常**：本地响应 200，配置正确，防火墙已放行
- **网络层被校园网阻断**：AP 隔离策略导致客户端间 TCP 握手无法完成
- **这不是 bug，是网络策略限制**：无法从电脑端修复，必须通过物理层绕行（热点/USB 共享）
- **已做优化**：修复了 `vite.config.ts` 中 `host: '127.0.0.1'` 的问题，服务已正确监听 `0.0.0.0`
